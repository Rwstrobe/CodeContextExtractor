# Security Policy

## Supported Versions
This project is currently in early development. Security fixes will be applied to the latest release only.

## Reporting a Vulnerability
If you discover a security issue, please open a GitHub issue with the label "security". If the report contains sensitive details, note that in the issue and maintainers will follow up to coordinate a private disclosure.

## Scope
This tool runs locally and makes no network calls. Reports should focus on issues such as unsafe file handling, unintended data exposure in output, or improper redaction behavior.
